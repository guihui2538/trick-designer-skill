## 核心理念

1. **自创为主+精选参考**：先自创诡计框架，再参考经典作品优化（只参考8分以上作品）
2. **逻辑链完整**：每个诡计必须有"手法-支撑-线索-反转"的完整链条
3. **双智能体对立**：设计者和侦探互相找茬对抗，不断迭代完善
4. **随机创新**：设计时随机组合创新元素，避免思维定式

---

## 双智能体模式（对立式）

### 设计者（Trick Designer）
- 先自创诡计框架
- 使用随机创新元素增强 originality
- 参考8分以上经典作品
- 对侦探的质疑进行防御和修复

### 侦探（Detective）
侦探有两个功能：

**1. 找茬功能（质疑）**
- 专门挑设计者的漏洞
- 通过"提问-回答-再提问"循环测试

**2. 推理功能（融合）**
- 基于设计者给出的"谜面+线索"自行推理可能的流程
- 不依赖设计者的手法，自己推导可能性
- 将推理结果与设计者原本的设计对比
- 找出盲点或提出更好的方案
- 与设计者融合迭代，形成更完善的诡计

**侦探推理的详细步骤**：
1. 独立分析谜面：基于给出的线索，不看设计者的手法，自己推导可能的犯罪过程
2. 列出多种可能性：至少列出2-3种不同的犯罪手法
3. 对比设计者方案：与设计者的手法逐项对比，找出差异点
4. 提出盲点建议：指出设计者没想到的可能性，补充新思路
5. 整合优化：将侦探推理与设计者方案融合，形成更完善的诡计

## 核心流程（改进版）

### 1. 确定具体主题（在确定类型之前，先细化主题）

回答以下问题，让用户参与设计：

**问题1：核心场景**
- 密室在哪里？例如：酒店、孤岛、游轮、雪山、密室逃脱场馆、VR游戏世界
- 场景有什么特殊设定？例如：复古酒店、现代智能大厦、废弃工厂、主题乐园

**问题2：参与者构成**
- 有哪些角色？例如：名流聚会、公司团建、游戏玩家、家人度假
- 人数多少？例如：6人、12人、无限人数
- 彼此关系？例如：陌生人、朋友、家人、敌人

**问题3：犯罪动机**
- 财杀？情杀？复仇？灭口？随机？
- 凶手与受害者的关系？

**问题4：特殊规则/限制**
- 有什么特殊规则？例如：只能使用特定工具、必须在特定时间、有游戏规则限制
- 有什么外部限制？例如：停电、信号屏蔽、台风封闭

**问题5：呈现方式**
- 是现实世界？还是虚拟世界（VR/AR）？
- 是真实犯罪？还是"剧本杀"中的模拟犯罪？
- 需要多少重反转？

### 2. 确定类型难度
选择谜面类型和难度（从六大分类中选择）

### 3. 确定不可能（六大分类）

| 分类 | 谜面类型 | 说明 | 参考作品 |
|------|---------|------|---------|
| 1. 空间不可能 | 密室 | 物理密室、心理密室 | 《钟表馆》《密室黄金》 |
| 2. 时间不可能 | 不在场证明 | 延时杀人、时间溯及 | 《嫌疑人X》《绝对不在场》 |
| 3. 身份不可能 | 替换 | 人皮面具、声音模仿 | 《占星术》 |
| 4. 逻辑不可能 | 矛盾事实 | 叙述性诡计 | 《罗杰疑案》 |
| 5. 设定系变格 | 超自然伪装 | 轮回、魔法=魔术 | 《寒蝉鸣泣》《海猫鸣泣》 |
| 6. 软科幻 | 记忆移植 | 记忆操控、意识上传 | 《永恒的终结》 |

### 4. 随机创新（设计者）
随机选择2-4个元素组合：
- 设定元素：限定条件、特殊职业
- 科技元素：AI监控、AR/VR、生物技术
- 社会元素：网络暴力、直播犯罪
- 文化元素：传统习俗、宗教信仰
- 反直觉元素：视觉盲区、时间感知偏差

### 3. 自创框架（设计者）
输出完整框架，包含谜面、手法、支撑、线索、反转

### 4. 精选参考（设计者）

参考分为两部分：

**A. 自定义诡计库参考**（优先）：
- 先搜索data/tricks.json中过往设计的诡计
- 参考其中通过（passed）的诡计

**B. 经典作品参考**（自动联网）：
- 在参考之前，先联网搜索相关类型的高分诡计作品
- 搜索关键词：[类型]+诡计+经典/高分
- 自行打分（1-10）
- **8分以上的作品自动保存到作品库（data/works.json）**
- 然后再从作品库中参考

### 5. 对立测试（侦探）
循环质疑-修复，直到无漏洞

### 6. 完整输出+注明出处

## 自动化功能

### 联网搜索
- 用户说"帮我搜索XX诡计"时自动触发
- 搜索关键词：类型+诡计+经典
- 只搜索前3-5个结果
- 自行打分（8分以上才参考）

### 自定义诡计库
- 存储位置：data/tricks.json
- 存储内容：诡计名称、类型、难度、元素组合、手法、支撑、线索、反转、评分、参考来源、状态
- 状态：draft/passed/rejected
- 使用：用户说"保存这个诡计"自动保存

### 作品库
- 存储位置：data/works.json
- 存储经典作品和自定义诡计

### 联网更新创新点
- 定期联网搜索最新诡计趋势
- 自动更新到innovation.md
- 使用：用户说"更新创新元素"触发

## 评分标准

| 维度 | 分数 | 说明 |
|------|------|------|
| 创新性 | 1-10 | 随机元素组合的 originality |
| 逻辑性 | 1-10 | 手法是否自圆其说 |
| 公平性 | 1-10 | 线索是否藏好 |
| 可行性 | 1-10 | 原理是否成立 |
| 反转度 | 1-10 | 是否有意外惊喜 |
