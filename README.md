# Trick Designer

> 推理小说诡计设计工具 | 兼容 Claude Code, OpenClaw, Codex, Gemini CLI

## 简介

Trick Designer 是一个专门用于设计推理小说诡计的工具。它帮助创作者从"不可能"的谜面出发，通过反直觉的心理学盲区或物理科学原理设计作弊点，并形成逻辑闭环。

该工具采用双智能体模式：AI扮演"设计者"负责构建诡计，扮演"侦探"负责破解和评测，通过对抗迭代不断优化诡计质量。

---

## 核心功能

### 双智能体模式
- **设计者（Trick Designer）**：负责诡计设计，无推理记忆，每次从零开始构建
- **侦探（Detective）**：负责推理分析，尝试破解设计者的诡计，通过反思优化
- 工作流程：设计者构建谜面 → 设计师计作弊点 → 侦探尝试破解 → 发现漏洞 → 设计者修复 → 迭代优化

### 难度分级
- **入门**：单一诡计、清晰线索，适合新手练手
- **进阶**：组合诡计、一到两处反转，适合中级创作
- **大师**：多重反转、叙诡结合，适合核心诡计

### 自动联网研究
用户输入诡计设定后，AI会自动：
1. 生成搜索关键词
2. 联网搜索相关作品
3. 分析作品，提取模式
4. 保存好的作品到经典作品库

### 作品库管理
- 经典作品库：保存看过的推理小说、游戏、动画
- 自定义诡计库：保存设计过的诡计
- 状态管理：draft（草稿）/ passed（通过）/ rejected（拒绝）

### 随机创新系统
随机组合5类元素创造新诡计：
- 设定元素：限定条件、特殊职业、时间/空间限制
- 科技元素：AI监控、AR/VR、生物技术、区块链
- 社会元素：网络暴力、直播犯罪、社交网络
- 文化元素：传统习俗、宗教信仰、游戏文化
- 反直觉元素：视觉盲区、时间感知偏差

## 支持的诡计类型

### 空间不可能
密室、不可能进入、瞬间移动
- 经典案例：《占星术杀人事件》《钟表馆事件》《斜屋犯罪》

### 时间不可能
完美不在场证明、延时犯罪
- 经典案例：《嫌疑人X的献身》《圣女的救济》《罗杰疑案》

### 身份不可能
双胞胎、假死、身份替换
- 经典案例：《分裂的推理》《GOTH断掌事件》

### 逻辑不可能
矛盾事实同时为真
- 经典案例：《罗杰疑案》《恶意》

### 设定系变格
超自然伪装、伪轮回、伪魔法
- 经典案例：《寒蝉鸣泣之时》《海猫鸣泣之时》《Another》《象山首》

### 软科幻
记忆移植、时间悖论、虚拟世界、平行世界
- 经典案例：《命运石之门》《Ever17》《流景之海·阿普利亚》

## 完整设计流程

### 第一步：确定不可能
用户选择谜面类型（6大类之一）

### 第二步：难度分级
选择入门/进阶/大师

### 第三步：选择方向
- 先参考经典诡计库
- 再选择作弊点方向（心理学或物理科学）

### 第四步：构建模板
根据类型选择具体模板（密室模板/不在场证明模板/身份模板）

### 第五步：反转设计
设计设定反转/身份反转/手法反转/动机反转

### 第六步：侦探测试
- 从侦探角度思考会怎么推理
- 找出推理路线上的明显线索
- 检验是否有反转到来的惊喜感
- 记录漏洞并修复

### 第七步：检验公平性
- 线索是否隐藏在文本中
- 是否有反转感
- 动机是否合理
- 是否避开常见思维定式

### 第八步：评分迭代
5个维度评分，根据总分决定是否通过

---

## 评分标准

| 维度 | 分数 | 说明 |
|------|------|------|
| 创新性 | 1-10 | 是否跳出常见模式 |
| 逻辑性 | 1-10 | 是否自圆其说 |
| 公平性 | 1-10 | 线索是否藏好 |
| 可行性 | 1-10 | 原理是否成立 |
| 反转度 | 1-10 | 是否有意外惊喜 |

### 决策

- 总分<30分：pass，重新生成
- 总分30-60分：优化后使用
- 总分>60分：优秀，使用

## 交互规则

1. **渐进式选择**：先选谜面类型，再选作弊点方向，最后选具体模板
2. **双智能体测试**：设计完让侦探尝试破解，再优化
3. **多方案选择**：必要时提供2-3个方案供用户选择
4. **作品库参考**：设计前先搜索已有作品和诡计
5. **自动保存**：设计完成后自动保存到作品库
6. **联网研究**：设计前必须联网搜索相关作品
7. **注明参考**：每个设计的诡计必须注明参考了哪些作品
