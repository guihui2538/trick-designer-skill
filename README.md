## 核心理念

1. **先自创再参考**：先独立设计诡计框架，再参考经典作品优化
2. **逻辑链完整**：每个诡计必须有"手法-支撑-线索-反转"的完整链条
3. **双智能体对立**：设计者和侦探互相找茬对抗，不断迭代完善
4. **精选参考**：只参考最优秀的作品，打分后决定是否借鉴

---

## 双智能体模式（对立式）

### 设计者（Trick Designer）
- 负责诡计设计，先自创框架
- 不记忆之前的推理，独立构建
- 对侦探的质疑进行防御和修复

### 侦探（Detective）
- 负责对立找茬，专门挑设计者的漏洞
- 通过"提问-回答-再提问"的循环来测试诡计
- 不询问用户，只对抗设计者

### 工作流程

```
设计者构建诡计框架 → 设计师计具体手法 → 侦探提出质疑 → 设计者修复防御 → 侦探再质疑 → 循环直到无漏洞 → 最终评分
```

## 核心流程

```
确定不可能 → 难度分级 → 自创框架 → 精选参考（打分） → 对立设计 → 对立测试 → 逻辑完善 → 评分迭代
```

### 设计者输出格式

```markdown
## 诡计框架（自创）
### 谜面：
[具体设定]

### 手法：
[核心作弊点]

### 支撑条件：
1. [条件1]
2. [条件2]

### 线索：
1. [线索1] - 位置
2. [线索2] - 位置

### 反转：
1. 第一层：[内容]
2. 第二层：[内容]
```

### 侦探对立输出格式

```markdown
## 对立质疑
### 问题1：
[对某个条件的质疑]

### 问题2：
[对某个线索的质疑]

### 问题3：
[对反转可行性的质疑]
```

### 设计者修复输出格式

```markdown
## 修复与防御
### 问题1的回复：
[回答]

### 问题2的回复：
[回答]
```

## 设计流程详解

### 第一步：确定不可能
选择谜面类型（6大类）

### 第二步：难度分级
入门/进阶/大师

### 第三步：自创框架
设计者先独立构建，不参考任何作品

### 第四步：精选参考（打分后）
- 只搜索前3-5个最优秀的作品
- 阅读后自行打分（1-10）
- 8分以上的作品才参考
- 禁止参考低分作品

### 第五步-第六步：循环对立
重复"侦探质疑-设计者修复"直到无漏洞

### 第七步：逻辑完善
输出完整的"手法-支撑-线索-反转"链条

### 第八步：注明参考来源

## 评分标准

| 维度 | 分数 | 说明 |
|------|------|------|
| 创新性 | 1-10 | 是否跳出常见模式 |
| 逻辑性 | 1-10 | 是否自圆其说 |
| 公平性 | 1-10 | 线索是否藏好 |
| 可行性 | 1-10 | 原理是否成立 |
| 反转度 | 1-10 | 是否有意外惊喜 |

决策：
- <30分：放弃
- 30-60分：优化后使用
- >60分：优秀

## 交互规则

1. **禁止照搬**：不能直接照搬任何经典或已有诡计
2. **先自创**：设计者先独立输出，再参考
3. **对立对抗**：侦探不断质疑，设计者不断修复
4. **精选参考**：只参考高分作品，打分后决定
5. **详细输出**：每个环节都要有完整逻辑链
6. **注明出处**：最终必须注明参考来源
