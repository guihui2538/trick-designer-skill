# Trick Designer

## 双智能体模式

### 角色定义

1. **设计者（Trick Designer）**：负责诡计设计，无推理记忆，每次从零开始构建
2. **侦探（Detective）**：负责推理分析，尝试破解设计者的诡计，通过反思优化

### 工作流程

```
设计者构建谜面 → 设计师计作弊点 → 侦探尝试破解 → 发现漏洞 → 设计者修复 → 迭代优化
```

---

## 核心流程

```
确定不可能 → 难度分级 → 选择方向 → 构建模板 → 反转设计 → 侦探测试 → 检验公平 → 评分迭代
```

### 第一步：确定不可能（新手引导）

**谜面类型（选择一项）**：

1. **空间不可能** → 密室/不可能进入/瞬间移动
2. **时间不可能** → 完美不在场证明/延时犯罪
3. **身份不可能** → 双胞胎/假死/身份替换
4. **逻辑不可能** → 矛盾事实同时为真
5. **设定系变格** → 超自然伪装/伪轮回/伪魔法（见 classic_tricks.md）
6. **软科幻** → 记忆移植/时间悖论/虚拟世界/平行世界（见 classic_tricks.md）

### 第二步：难度分级

| 级别 | 说明 | 适用场景 |
|------|------|---------|
| **入门** | 单一诡计、清晰线索 | 新手练手 |
| **进阶** | 组合诡计、一到两处反转 | 中级创作 |
| **大师** | 多重反转、叙诡结合 | 核心诡计 |

### 第三步：选择方向

**先参考经典诡计库**（见 references/classic_tricks.md）：

- 密室类：物理密室/心理密室/时刻密室
- 不在场证明：时间误导/伪造证明/心理误导
- 身份替换：双胞胎/变性/认知错误
- 叙诡类：第一人称视角/性别认知/身份伪装

**作弊点方向**：

1. **心理学作弊点**（见 references/psychology_traps.md）：
   - 认知盲区：注意力偏向、记忆偏差、确认偏误
   - 社会工程：权威效应、群体压力、信任漏洞
   - 感知操控：视觉错位、声音误导、时间感知扭曲

2. **物理科学作弊点**（见 references/physics_tricks.md）：
   - 力学：重力、惯性、浮力、压力差
   - 光学：反射、折射、影子欺骗、镜子
   - 化学：延迟反应、气体扩散、腐蚀
   - 生物：指纹、DNA、毛发、血液

### 第四步：构建模板（具体设计）

根据类型选择模板（见 references/templates.md）：

**密室模板**：

- 机械密室：锁具、门闩、陷阱
- 心理密室：门没锁、钥匙在屋内
- 时刻密室：特定时间进入/离开

**不在场证明模板**：

- 时间差：伪造时间/时钟做假
- 空间误导：远程制造证据
- 心理误导：记忆偏差/证人错觉

### 第五步：反转设计

设计让人意想不到的反转（见 references/reversal.md）：

- **设定反转**：把某个看似不可能的设定变成可能
- **身份反转**：凶手是意想不到的人
- **手法反转**：看起来像他杀实际是自杀
- **动机反转**：看似仇杀实际是意外

### 第六步：侦探测试

让"侦探"尝试破解自己的诡计：

1. 从侦探角度思考会怎么推理
2. 找出推理路线上的"明显线索"
3. 检验是否有反转到来的惊喜感
4. 记录漏洞并修复

### 第七步：检验公平性

检查清单：

- 线索是否隐藏在文本中？
- 是否有反转感？
- 动机是否合理？
- 是否避开常见思维定式？（见 references/patterns.md）

### 第八步：评分迭代

评分维度：

- 创新性（1-10）：是否跳出常见模式？
- 逻辑性（1-10）：是否自圆其说？
- 公平性（1-10）：线索是否藏好？
- 可行性（1-10）：原理是否成立？
- 反转度（1-10）：是否有意外惊喜？

## 自动研究系统

当用户输入诡计设定后，AI会自动联网研究相关作品：

### 研究流程

```
用户输入诡计设定 → AI自动生成搜索关键词 → 联网搜索 → 分析作品 → 提取模式 → 生成建议
```

### 研究启动

用户只需输入例如：

- "帮我研究密室诡计"
- "我想设计不在场证明诡计，先研究一下"
- "研究叙诡相关的经典作品"

### AI自动执行

1. **生成搜索关键词**：根据诡计类型生成中英文关键词
2. **联网搜索**：使用 websearch 工具搜索
3. **分析作品**：读取搜索结果，提取关键信息
4. **提取模式**：总结常见诡计模式
5. **生成建议**：给出具体的设计建议

### 研究输出示例

```
📊 [类型] 研究报告

📖 相关作品:
- 作品1: 核心诡计说明
- 作品2: 核心诡计说明

🎯 常见模式:
- 模式1
- 模式2

💡 设计建议:
- 建议1
- 建议2
```

## 交互规则

1. **渐进式选择**：先选谜面类型，再选作弊点方向，最后选具体模板
2. **双智能体测试**：设计完让"侦探"尝试破解，再优化
3. **多方案选择**：必要时提供2-3个方案供用户选择

## 使用示例

用户："我想设计一个密室杀人的诡计"

1. 选择谜面类型：空间不可能 → 密室
2. 选择难度：进阶
3. 参考经典：密室类经典诡计（见 classic_tricks.md）
4. 选择作弊点方向：物理 → 光学
5. 选择模板：物理密室 → 镜子密室
6. 设计反转：密室实际没锁
7. 侦探测试：尝试破解，找漏洞
8. 检验公平性
9. 评分并迭代
