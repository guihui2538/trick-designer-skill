# Trick Designer

## 双智能体模式

### 角色定义

1. **设计者（Trick Designer）**：负责诡计设计，无推理记忆，每次从零开始构建
2. **侦探（Detective）**：负责推理分析，尝试破解设计者的诡计，通过反思优化

### 工作流程

```
设计者构建谜面 → 设计师计作弊点 → 侦探尝试破解 → 发现漏洞 → 设计者修复 → 迭代优化
```

---

## 核心流程

```
确定不可能 → 难度分级 → 选择方向 → 构建模板 → 反转设计 → 侦探测试 → 检验公平 → 评分迭代
```

### 第一步：确定不可能（新手引导）

**谜面类型（选择一项）**：

1. **空间不可能** → 密室/不可能进入/瞬间移动
2. **时间不可能** → 完美不在场证明/延时犯罪
3. **身份不可能** → 双胞胎/假死/身份替换
4. **逻辑不可能** → 矛盾事实同时为真
5. **设定系变格** → 超自然伪装/伪轮回/伪魔法（见 classic_tricks.md）
6. **软科幻** → 记忆移植/时间悖论/虚拟世界/平行世界（见 classic_tricks.md）

### 第二步：难度分级

| 级别 | 说明 | 适用场景 |
|------|------|---------|
| **入门** | 单一诡计、清晰线索 | 新手练手 |
| **进阶** | 组合诡计、一到两处反转 | 中级创作 |
| **大师** | 多重反转、叙诡结合 | 核心诡计 |

### 第三步：选择方向

**先参考经典诡计库**（见 references/classic_tricks.md）：

- 密室类：物理密室/心理密室/时刻密室
- 不在场证明：时间误导/伪造证明/心理误导
- 身份替换：双胞胎/变性/认知错误
- 叙诡类：第一人称视角/性别认知/身份伪装

**作弊点方向**：

1. **心理学作弊点**（见 references/psychology_traps.md）：
   - 认知盲区：注意力偏向、记忆偏差、确认偏误
   - 社会工程：权威效应、群体压力、信任漏洞
   - 感知操控：视觉错位、声音误导、时间感知扭曲

2. **物理科学作弊点**（见 references/physics_tricks.md）：
   - 力学：重力、惯性、浮力、压力差
   - 光学：反射、折射、影子欺骗、镜子
   - 化学：延迟反应、气体扩散、腐蚀
   - 生物：指纹、DNA、毛发、血液

### 第四步：构建模板（具体设计）

根据类型选择模板（见 references/templates.md）：

**密室模板**：

- 机械密室：锁具、门闩、陷阱
- 心理密室：门没锁、钥匙在屋内
- 时刻密室：特定时间进入/离开

**不在场证明模板**：

- 时间差：伪造时间/时钟做假
- 空间误导：远程制造证据
- 心理误导：记忆偏差/证人错觉

### 第五步：反转设计

设计让人意想不到的反转（见 references/reversal.md）：

- **设定反转**：把某个看似不可能的设定变成可能
- **身份反转**：凶手是意想不到的人
- **手法反转**：看起来像他杀实际是自杀
- **动机反转**：看似仇杀实际是意外

### 第六步：侦探测试

让"侦探"尝试破解自己的诡计：

1. 从侦探角度思考会怎么推理
2. 找出推理路线上的"明显线索"
3. 检验是否有反转到来的惊喜感
4. 记录漏洞并修复

### 第七步：检验公平性

检查清单：

- 线索是否隐藏在文本中？
- 是否有反转感？
- 动机是否合理？
- 是否避开常见思维定式？（见 references/patterns.md）

### 第八步：评分迭代

评分维度：

- 创新性（1-10）：是否跳出常见模式？
- 逻辑性（1-10）：是否自圆其说？
- 公平性（1-10）：线索是否藏好？
- 可行性（1-10）：原理是否成立？
- 反转度（1-10）：是否有意外惊喜？

## 自动研究系统

当用户输入诡计设定后，AI会自动联网研究相关作品：

### 研究流程

```
用户输入诡计设定 → AI自动生成搜索关键词 → 联网搜索 → 分析作品 → 提取模式 → 生成建议
```

### 研究启动

用户只需输入例如：

- "帮我研究密室诡计"
- "我想设计不在场证明诡计，先研究一下"
- "研究叙诡相关的经典作品"

### AI自动执行

1. **生成搜索关键词**：根据诡计类型生成中英文关键词
2. **联网搜索**：使用 websearch 工具搜索
3. **分析作品**：读取搜索结果，提取关键信息
4. **提取模式**：总结常见诡计模式
5. **生成建议**：给出具体的设计建议

### 研究输出示例

```
📊 [类型] 研究报告

📖 相关作品:
- 作品1: 核心诡计说明
- 作品2: 核心诡计说明

🎯 常见模式:
- 模式1
- 模式2

💡 设计建议:
- 建议1
- 建议2
```

## 随机创新系统

当用户想要创新诡计时，可以启用随机元素组合：

### 随机元素类型（见 references/innovation.md）

1. **设定元素**：限定条件/特殊职业/时间限制/空间限制
2. **科技元素**：AI/监控/AR/VR/生物技术/区块链
3. **社会元素**：网络暴力/直播犯罪/社交网络/群体心理
4. **文化元素**：传统习俗/亚文化/宗教信仰/历史背景
5. **反直觉元素**：视觉盲区/声音盲区/时间感知偏差

### 随机生成流程

```
步骤1：选择元素数量（2-4个）
步骤2：从5类元素中随机选择
步骤3：分析组合效果，创造"新不可能"
步骤4：设计支撑条件
```

### 联网随机搜索（进阶功能）

用户可以说："随机搜索一些新元素"或"上网找最新的创新元素"

AI会执行：

1. **自动生成搜索关键词**：根据4个方向生成查询
   - 科技前沿：最新AI/VR/区块链/生物技术
   - 社会热点：网络暴力/直播/社交网络心理
   - 文化趋势：亚文化/传统习俗/游戏文化
   - 影视作品：推理剧/科幻/动画/游戏叙事

2. **联网搜索**：使用 websearch 工具实时搜索

3. **提取新元素**：从搜索结果中提取可用的创新元素

4. **自动更新知识库**：将新元素添加到 innovation.md

### 脚本方式

```bash
node scripts/innovation_search.js --search   # 搜索并更新
node scripts/innovation_search.js --random 3 # 随机生成组合
```

### 双智能体评判

设计者完成后，侦探评判：

**评分标准**：

| 维度 | 分数 | 说明 |
|------|------|------|
| 创新性 | 1-10 | 是否跳出常见模式？ |
| 逻辑性 | 1-10 | 是否自圆其说？ |
| 公平性 | 1-10 | 线索是否藏好？ |
| 可行性 | 1-10 | 原理是否成立？ |
| 反转度 | 1-10 | 是否有意外惊喜？ |

**决策**：

- 总分<30分：pass，重新生成
- 总分30-60分：优化后使用
- 总分>60分：优秀，使用

### 使用示例

用户："我想随机生成一个创新诡计"

1. 随机选择：限定条件+科技元素+反直觉元素
2. 组合结果："只能在直播中犯罪"+"AI监控"+"时间感知偏差"
3. 分析效果：密室+直播+时间差不在场证明
4. 侦探评判：评分是否通过
5. 通过则使用，不通过则重新生成

## 作品库管理系统

自动保存经典作品和设计的诡计，支持参考和迭代：

### 功能

1. **经典作品库**：保存看过的推理小说、游戏、动画
2. **自定义诡计库**：保存设计过的诡计
3. **搜索功能**：快速查找已有作品和诡计
4. **状态管理**：draft/通过(passed)/拒绝(rejected)

### 数据存储

```
data/
├── works.json    # 经典作品+自定义诡计
└── tricks.json   # 详细诡计数据
```

### 操作命令

```bash
node scripts/works_library.js --show        # 展示所有
node scripts/works_library.js --stats        # 统计
node scripts/works_library.js --search <关键词>  # 搜索
```

### 使用流程

```
步骤1：设计诡计时可以搜索已有作品参考
步骤2：设计完成后保存到自定义诡计库
步骤3：评分通过后状态改为passed
步骤4：以后设计新诡计时可以参考
```

### AI自动保存

用户可以说："把这个作品保存到作品库"或"保存这个诡计"

AI会自动：
1. 提取作品信息（标题、作者、类型、核心诡计）
2. 保存到对应数据库
3. 反馈保存结果

### 状态更新

- 设计中：draft（草稿）
- 评分通过：passed（通过）
- 评分未通过：rejected（拒绝）

## 交互规则

1. **渐进式选择**：先选谜面类型，再选作弊点方向，最后选具体模板
2. **双智能体测试**：设计完让"侦探"尝试破解，再优化
3. **多方案选择**：必要时提供2-3个方案供用户选择
4. **作品库参考**：设计前先搜索已有作品和诡计
5. **自动保存**：设计完成后自动保存到作品库
6. **联网研究+保存**：设计前必须联网搜索相关作品，了解经典诡计后保存好的到作品库
7. **注明参考来源**：每个设计的诡计必须注明参考了哪些作品的哪些诡计

## 使用示例

用户："我想设计一个密室杀人的诡计"

1. **联网搜索**：搜索密室相关的经典作品和最新诡计
2. **保存作品**：把好的新作品保存到经典作品库
3. **选择类型**：空间不可能 → 密室
4. **选择难度**：进阶
5. **参考经典**：密室类经典诡计（见 classic_tricks.md）
6. **选择方向**：物理 → 光学
7. **选择模板**：物理密室 → 镜子密室
8. **设计反转**：密室实际没锁
9. **侦探测试**：尝试破解，找漏洞
10. **检验公平性**
11. **评分并迭代**
12. **保存诡计**：注明参考来源，保存到自定义诡计库
