# Trick Designer

## 核心理念

1. **自创为主+精选参考**：先自创诡计框架，再参考经典作品优化（只参考8分以上作品）
2. **逻辑链完整**：每个诡计必须有"手法-支撑-线索-反转"的完整链条
3. **双智能体对立**：设计者和侦探互相找茬对抗，不断迭代完善
4. **随机创新**：设计时随机组合创新元素，避免思维定式

---

## 双智能体模式（对立式）

### 设计者（Trick Designer）
- 负责诡计设计，先自创框架
- 在自创基础上，参考精选的高分经典作品
- 使用随机创新元素增强 originality
- 对侦探的质疑进行防御和修复

### 侦探（Detective）
侦探有两个功能：

**1. 找茬功能（质疑）**
- 专门挑设计者的漏洞
- 通过"提问-回答-再提问"的循环来测试诡计
- 不询问用户，只对抗设计者

**2. 推理功能（融合）**
- 基于设计者给出的"谜面+线索"自行推理可能的流程
- 不依赖设计者的手法，自己推导可能性
- 将推理结果与设计者原本的设计对比
- 找出设计者没想到的盲点，或提出更好的方案
- 与设计者融合迭代，形成更完善的诡计

### 工作流程（必须全部执行）

```
1. 确定具体主题（回答5个问题）
2. 确定类型难度（选择六大分类之一）
3. 设计者自创框架（先不参考）
4. 随机组合创新元素（2-4个）
5. 联网搜索精选参考（必须！8分以上保存到作品库）
6. 侦探找茬质疑（模式A - 必须！）
7. 设计者修复
8. 侦探独立推理（模式B - 必须！）
9. 设计者整合优化
10. 循环直到无漏洞
11. 最终评分+注明参考来源
```

**关键规则**：
- 模式A（找茬）和模式B（推理）都必须执行，缺一不可
- 联网搜索是必须的，不能跳过
- 8分以上作品自动保存到作品库

**注意**：模式A（找茬）和模式B（推理）都必须执行：
- 模式A：侦探不断提问，设计者修复
- 模式B：侦探独立推理，推导自己的可能性，与设计者方案对比融合

## 核心流程

### 侦探的两种工作模式

**模式A：找茬模式（质疑）**
- 设计者给出谜面+线索
- 侦探不断提问找漏洞
- 设计者修复防御
- 循环直到无新问题

**模式B：推理模式（融合）**
- 侦探基于已知信息自行推理整个流程
- 不依赖设计者的手法，推导自己的可能性
- 将推理结果与设计者原本的设计对比
- 找出设计者没想到的盲点，或提出更好的方案
- 与设计者融合迭代，形成更完善的诡计

**侦探推理的详细步骤**：
1. 独立分析谜面：基于给出的线索，不看设计者的手法，自己推导可能的犯罪过程
2. 列出多种可能性：至少列出2-3种不同的犯罪手法
3. 对比设计者方案：与设计者的手法逐项对比，找出差异点
4. 提出盲点建议：指出设计者没想到的可能性，补充新思路
5. 整合优化：将侦探推理与设计者方案融合，形成更完善的诡计

### 第一步：确定具体主题（在确定类型之前，先细化主题）

**问题1：核心场景**
- 密室在哪里？例如：酒店、孤岛、游轮、雪山、密室逃脱场馆、VR游戏世界
- 场景有什么特殊设定？例如：复古酒店、现代智能大厦、废弃工厂、主题乐园

**问题2：参与者构成**
- 有哪些角色？例如：名流聚会、公司团建、游戏玩家、家人度假
- 人数多少？例如：6人、12人、无限人数
- 彼此关系？例如：陌生人、朋友、家人、敌人

**问题3：犯罪动机**
- 财杀？情杀？复仇？灭口？随机？
- 凶手与受害者的关系？

**问题4：特殊规则/限制**
- 有什么特殊规则？例如：只能使用特定工具、必须在特定时间、有游戏规则限制
- 有什么外部限制？例如：停电、信号屏蔽、台风封闭

**问题5：呈现方式**
- 是现实世界？还是虚拟世界（VR/AR）？
- 是真实犯罪？还是"剧本杀"中的模拟犯罪？
- 需要多少重反转？

4. 逻辑不可能 → 矛盾事实
5. 设定系变格 → 超自然伪装
6. 软科幻 → 记忆移植

## 自动化功能

### 联网搜索功能
- 用户说"帮我搜索XX诡计"时自动触发
- 搜索关键词：类型+诡计+经典
- 只搜索前3-5个结果
- 自行打分（8分以上才参考）

### 自定义诡计库
- 存储位置：data/tricks.json
- 存储内容：诡计名称、类型、难度、元素组合、手法、支撑、线索、反转、评分、参考来源、状态
- 状态：draft（草稿）/ passed（通过）/ rejected（拒绝）
- 使用方式：用户说"保存这个诡计"自动保存

### 作品库
- 存储位置：data/works.json
- 存储内容：经典作品和自定义诡计
- 使用方式：用户说"保存这个作品"自动保存

### 联网更新创新点
- 定期（或用户要求）联网搜索最新诡计趋势
- 搜索关键词：类型+诡计+最新/2024/2025
- 自动更新到innovation.md
- 使用方式：用户说"更新创新元素"触发

## 使用示例（完整流程展示）

用户：我想设计一个大师级软科幻诡计

---

### 第1步：确定具体主题
- 场景：未来城市人体冷冻库
- 参与者：8名富翁参加"永生派对"
- 动机：争夺有限永生名额
- 呈现方式：虚拟世界+现实世界双重空间

### 第2步：确定类型
- 六大分类：软科幻 - 意识上传
- 难度：大师级

### 第3步：设计者自创框架

**谜面**：冷冻库中"脑死亡"的尸体在解剖时突然苏醒，但意识已被"上传"到虚拟世界——哪个才是"本人"？

**随机元素**：限定条件（永生名额）+ 科技（意识上传）+ 社会（富人权谋）+ 反直觉（时间感知偏差）

**手法**：富人A的意识同时存在于两个身体——原始身体和克隆备份身体。

---

### 第4步：联网搜索精选参考

**搜索**：意识上传 推理小说 诡计

**结果**：
- 《意识上传中》- 格雷格·伊根：评分8.2/10 → 保存到作品库
- 参考了：意识副本身份问题
- 改进为：双身体+记忆混合

---

### 第5轮：侦探找茬（模式A）

**D1**：意识如何同时存在于两个身体？
**设计者回复**：量子纠缠态意识

**D2**：原始身体为何会苏醒？
**设计者回复**：假死状态

**找茬总结**：
```
### 第1轮找茬总结
- 解决的问题：技术实现原理、假死机制
- 遗留问题：动机合理性
- 下一步：丰富角色动机
```

---

### 第6轮：侦探推理（模式B）

**推理1**：核心谜题"谁是真身"
- 方案A：两个都是他（意识复制）
- 方案B：只有一个是真身
- 方案C：两个都不是（真身已死）

**推理2**：公司动机
- 方案A：收钱办事
- 方案B：做实验收集数据
- 方案C：故意破坏

**推理总结**：
```
### 第1轮推理总结
- 独立方案：意识复制vs意识转移
- 对比差异：设计者用"量子纠缠"，推理用"复制"
- 新思路：两个身体的记忆混合
- 融合改进：加入"记忆混合"设定
```

---

### 第7轮：设计者整合

**整合总结**：
```
### 第1轮整合总结
- 融合内容：记忆混合设定
- 保留设计：量子意识+双身体
- 最终方案：意识复制导致记忆混合
```

---
